# Rakta.js Code of Conduct

---

## Indonesian

Rakta.js dibangun untuk menjadi project open source yang sehat, terbuka, dan nyaman untuk dikembangkan bersama.

Code of Conduct ini berlaku untuk semua orang yang berpartisipasi di project Rakta.js, baik sebagai maintainer, contributor, user, reviewer, maupun orang yang hanya ikut berdiskusi di issue, Pull Request, atau channel komunitas.

---

### Komitmen Kami

Sebagai maintainer dan contributor Rakta.js, kami berkomitmen untuk menjaga lingkungan kontribusi yang terbuka, ramah, dan bebas dari pelecehan.

Kami ingin Rakta.js menjadi tempat yang aman untuk:
- developer yang baru pertama kali membuat Pull Request
- developer yang sedang belajar open source
- developer yang memakai Rakta.js untuk project pribadi
- developer yang memakai Rakta.js untuk project production
- maintainer yang menjaga kualitas project
- contributor yang datang dari background, bahasa, dan level pengalaman berbeda

Kami menghargai kontribusi teknis, dokumentasi, diskusi, bug report, saran fitur, review kode, dan masukan yang membantu Rakta.js menjadi lebih baik.

---

### Standar Perilaku yang Diharapkan

Contoh perilaku yang membantu menjaga komunitas tetap sehat:
- Menggunakan bahasa yang sopan dan mudah dipahami.
- Menghormati sudut pandang dan pengalaman orang lain.
- Memberi kritik ke kode, bukan menyerang orangnya.
- Menerima feedback dengan terbuka.
- Menjelaskan masalah teknis dengan jelas.
- Membantu contributor baru memahami alur project.
- Menghargai waktu reviewer dan maintainer.
- Memberi kredit yang layak kepada contributor.
- Menghindari debat framework yang tidak produktif.
- Fokus pada kualitas project dan pengalaman user.

Kritik teknis boleh tegas, tapi tetap harus diarahkan ke masalahnya, bukan ke pribadi orang lain.

Contoh:
```txt
Bagian ini bisa dibuat lebih aman karena tipe response belum divalidasi.
```

Bukan:
```txt
Kode kamu jelek dan kamu tidak paham TypeScript.
```

---

### Perilaku yang Tidak Diterima

Contoh perilaku yang tidak diterima di project ini:
- Menghina, merendahkan, atau menyerang contributor lain.
- Melakukan pelecehan secara public atau private.
- Menggunakan komentar yang kasar, diskriminatif, atau merendahkan.
- Mengirim spam di issue, Pull Request, atau discussion.
- Membocorkan informasi pribadi orang lain tanpa izin.
- Membuat konflik framework, bahasa, atau tooling secara berlebihan.
- Mengganggu proses review dengan komentar yang tidak relevan.
- Memaksa maintainer menerima perubahan tanpa diskusi.
- Mengklaim pekerjaan orang lain sebagai pekerjaan sendiri.
- Menggunakan project space untuk promosi yang tidak berhubungan.
- Melakukan tindakan lain yang secara wajar dianggap tidak pantas dalam lingkungan open source.

Rakta.js terinspirasi oleh banyak framework dan tool. Perbandingan teknis boleh dilakukan, tapi “framework wars” tidak diterima di project ini.

---

### Tanggung Jawab Maintainer

Maintainer bertanggung jawab untuk:
- menjelaskan standar perilaku yang berlaku
- menjaga diskusi tetap sehat
- meninjau laporan pelanggaran dengan adil
- mengambil tindakan jika ada perilaku yang merugikan komunitas
- menjaga agar proses kontribusi tetap aman dan produktif

Maintainer boleh menghapus, menyembunyikan, atau menolak komentar, issue, Pull Request, commit, atau kontribusi lain yang melanggar Code of Conduct ini.

Maintainer juga boleh membatasi akses contributor yang terus mengganggu project.

---

### Ruang Lingkup

Code of Conduct ini berlaku di semua ruang project Rakta.js, termasuk:
- issue
- Pull Request
- discussion
- commit message
- komentar review
- dokumentasi
- channel komunitas resmi
- percakapan public ketika seseorang mewakili Rakta.js

Code of Conduct ini juga berlaku ketika seseorang menggunakan nama Rakta.js, logo Rakta.js, atau posisi sebagai maintainer/contributor Rakta.js di ruang public.

---

### Cara Melaporkan

Jika kamu melihat atau mengalami perilaku yang melanggar Code of Conduct ini, laporkan ke maintainer project.

Saat melaporkan, sertakan informasi yang jelas jika memungkinkan:
- link issue, Pull Request, atau discussion
- screenshot jika diperlukan
- waktu kejadian
- username yang terlibat
- ringkasan singkat masalah
- dampak yang terjadi

Laporan akan ditinjau dengan serius dan tidak akan digunakan untuk menyerang pelapor.

Jangan membuka public issue untuk masalah yang berisi informasi sensitif. Gunakan jalur private yang tersedia di repository.

---

### Tindakan Penegakan

Tindakan yang dapat diambil maintainer tergantung pada tingkat pelanggaran.

Beberapa tindakan yang mungkin dilakukan:
- memberi peringatan
- meminta perubahan bahasa atau sikap
- menyembunyikan komentar
- menutup issue atau Pull Request
- menolak kontribusi
- membatasi partisipasi sementara
- memblokir contributor dari project jika pelanggaran berat atau berulang

Tujuannya bukan untuk mempermalukan orang, tetapi untuk menjaga project tetap aman, sehat, dan produktif.

---

### Catatan Tentang Kritik Teknis

Rakta.js menerima kritik teknis.

Kamu boleh mengatakan:
```txt
API ini terlalu membingungkan.
```

```txt
Function ini sulit dites.
```

```txt
Struktur ini bisa menyebabkan bug di route dinamis.
```

Tapi jangan menyerang orangnya.

Tidak boleh:
```txt
Kamu bodoh.
```

```txt
Maintainer tidak becus.
```

```txt
Contributor ini tidak pantas ikut project.
```

Fokus ke kode, behavior, dokumentasi, dan pengalaman user.

## English

Rakta.js is built to be a healthy, open, and welcoming open source project.

This Code of Conduct applies to everyone participating in the Rakta.js project, including maintainers, contributors, users, reviewers, and people joining discussions in issues, Pull Requests, or community channels.

---

### Our Pledge

As maintainers and contributors of Rakta.js, we pledge to keep the contribution environment open, respectful, and free from harassment.

We want Rakta.js to be a safe place for:
- developers opening their first Pull Request
- developers learning open source
- developers using Rakta.js for personal projects
- developers using Rakta.js in production
- maintainers protecting project quality
- contributors from different backgrounds, languages, and experience levels

We value technical contributions, documentation, discussions, bug reports, feature requests, code reviews, and feedback that help Rakta.js become better.

---

### Expected Behavior

Examples of behavior that helps keep the community healthy:
- Using respectful and clear language.
- Respecting different viewpoints and experiences.
- Criticizing code, not the person who wrote it.
- Accepting feedback with an open attitude.
- Explaining technical problems clearly.
- Helping new contributors understand the project flow.
- Respecting the time of reviewers and maintainers.
- Giving proper credit to contributors.
- Avoiding unproductive framework debates.
- Focusing on project quality and user experience.

Technical criticism can be direct, but it must be aimed at the issue, not at the person.

Example:
```txt
This part can be made safer because the response type is not validated yet.
```

Not:
```txt
Your code is bad and you do not understand TypeScript.
```

---

### Unacceptable Behavior

Examples of unacceptable behavior in this project include:
- Insulting, mocking, or attacking another contributor.
- Public or private harassment.
- Using rude, discriminatory, or degrading comments.
- Spamming issues, Pull Requests, or discussions.
- Sharing someone else's private information without permission.
- Creating excessive framework, language, or tooling conflicts.
- Disrupting the review process with irrelevant comments.
- Pressuring maintainers to accept changes without discussion.
- Claiming someone else's work as your own.
- Using project spaces for unrelated promotion.
- Any other behavior that would reasonably be considered inappropriate in an open source environment.

Rakta.js is inspired by many frameworks and tools. Technical comparison is welcome, but framework wars are not welcome in this project.

---

### Maintainer Responsibilities

Maintainers are responsible for:
- clarifying the expected behavior
- keeping discussions healthy
- reviewing violation reports fairly
- taking action when behavior harms the community
- keeping the contribution process safe and productive

Maintainers may remove, hide, or reject comments, issues, Pull Requests, commits, or other contributions that violate this Code of Conduct.

Maintainers may also limit access for contributors who repeatedly disrupt the project.

---

### Scope

This Code of Conduct applies in all Rakta.js project spaces, including:
- issues
- Pull Requests
- discussions
- commit messages
- review comments
- documentation
- official community channels
- public conversations when someone is representing Rakta.js

This Code of Conduct also applies when someone uses the Rakta.js name, Rakta.js logo, or their position as a Rakta.js maintainer/contributor in public spaces.

---

### Reporting

If you see or experience behavior that violates this Code of Conduct, report it to the project maintainer.

When reporting, include clear information when possible:
- issue, Pull Request, or discussion link
- screenshot if needed
- time of the incident
- usernames involved
- short summary of the problem
- impact of the incident

Reports will be reviewed seriously and will not be used to attack the person who reported them.

Do not open a public issue for problems that contain sensitive information. Use the private reporting path available in the repository.

---

### Enforcement

Maintainer action depends on the severity of the violation.

Possible actions include:
- giving a warning
- asking for language or behavior changes
- hiding comments
- closing issues or Pull Requests
- rejecting contributions
- temporarily limiting participation
- blocking a contributor from the project for serious or repeated violations

The goal is not to shame people. The goal is to keep the project safe, healthy, and productive.

---

### Note About Technical Criticism

Rakta.js accepts technical criticism.

You may say:
```txt
This API is confusing.
```

```txt
This function is hard to test.
```

```txt
This structure may cause bugs in dynamic routes.
```

But do not attack the person.

Do not say:
```txt
You are stupid.
```

```txt
The maintainer is incompetent.
```

```txt
This contributor does not deserve to be in the project.
```

Focus on code, behavior, documentation, and user experience.
