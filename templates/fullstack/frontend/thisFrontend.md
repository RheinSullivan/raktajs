# Rakta.js fullstack frontend template

This frontend template mirrors the files generated by `create-rakta-app`
when **Fullstack app** is selected.

## Included systems

| System | Files generated |
| --- | --- |
| App router | `app/page.tsx`, auth pages, dashboard page, nested pages, API routes |
| Layout | `app/layout.tsx` |
| Loading/error/not found | `app/loading.tsx`, `app/error.tsx`, `app/not-found.tsx` |
| HTTP helper | `lib/http.ts` |
| Route constants | `lib/routes.ts` |
| Store | `stores/counter.store.ts` |
| Schema | `schemas/user.schema.ts` |
| Public assets | `public/favicon.ico` |

## Auth pages

- `app/auth/login/page.tsx`
- `app/auth/register/page.tsx`
- `app/auth/forgot-password/page.tsx`
- `app/auth/reset-password/page.tsx`
- `app/dashboard/page.tsx`

## Auto import

Generated Rakta apps are configured for framework auto import. Component
files should prefer Rakta public tags such as `<click to="/">` and
`<photo path="/image.svg">` instead of manual imports for framework
primitives.

## Backend connection

Set the backend URL with:

```env
API_URL=http://localhost:4000
```

The backend exposes `/api/hello` and `/api/auth/*` endpoints.
